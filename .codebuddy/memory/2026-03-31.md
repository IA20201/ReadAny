## 修复 iOS WebView fetch file:// 失败 (Failed to fetch: 0)
- **根因排查**：main 分支上 loadBook 本来就是用 base64 一次性传输（`FileSystem.readAsStringAsync` + `bridge.openBook({ base64 })`），是正常工作的
- 之前误以为需要改成 file URI 方案，结果引入了 `fetch('file://...')` 在 WKWebView 中不支持的问题
- 分块传输方案（base64Chunk/base64End）也可能引入时序问题
- **最终方案**：回退 loadBook 到 main 分支的原始实现——直接用 base64 一次性传输
- 移除了不必要的 `allowingReadAccessToURL="/"` WebView 属性
- `reader.template.html` 的 uri 分支也恢复为原始 `fetch()` 逻辑
- `use-reader-bridge.ts` 中保留了 `openBookChunked` 方法和 `reader.template.html` 中的 `base64Chunk/base64End` 处理（不影响正常功能，只是不会被调用）

## 修复点击翻页失效
- 根因：`reader.template.html` 的 `attachTapListener` 中只处理了中间区域(25%-75%)的点击（发 tap 事件切换工具栏），左右两侧区域(0-25%, 75%-100%)的点击没有任何处理，导致点击翻页完全无效
- 修复：在 `attachTapListener` 的 touchend 事件中添加三区域判断：
  - 左侧 <25%：调用 `window.goPrev()` 翻上一页
  - 右侧 >75%：调用 `window.goNext()` 翻下一页
  - 中间 25%-75%：发送 `tap` 事件（切换工具栏）
- 重新构建 reader.html
