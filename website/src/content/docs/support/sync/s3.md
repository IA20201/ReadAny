ReadAny supports S3-compatible object storage for long-term sync across devices.

## What You Need

In **Settings → Sync → S3**, you will usually fill:

- **Endpoint**
- **Region**
- **Bucket**
- **Access Key ID**
- **Secret Access Key**
- **Path-style** toggle when required by your provider

[Image path: s3-sync-settings]

## Field Guide

### Endpoint

The S3-compatible service URL, for example:

- `https://<account>.r2.cloudflarestorage.com`
- `https://s3.cn-east-1.qiniucs.com`
- `https://cos.ap-guangzhou.myqcloud.com`

### Region

Use the region required by your provider.

- some providers use real AWS-style regions
- some providers use `auto`

### Bucket

The bucket/container name where ReadAny will store sync data.

### Path-style

Enable this only if your provider requires path-style S3 URLs.

If the same credentials work on one platform but not another, this is one of the first settings to double-check.

[Image path: s3-provider-example-config]

## What S3 Sync Stores

S3 sync can store:

- device sync snapshots
- book metadata
- notes and reading stats
- covers
- book files

## Recommended Workflow

1. Configure S3 on one device
2. Test the connection
3. Save the config
4. Trigger a manual sync
5. Configure the second device with the same bucket
6. Sync again or use full download if needed

## Full Upload and Full Download

If you want to replace one side completely:

- use **Full Upload** on the device with the correct library
- use **Full Download** on the target device

This is often the fastest way to recover from a messy first sync.

## Troubleshooting

### Test connection fails

Check:

- endpoint
- region
- bucket name
- Access Key ID / Secret Access Key
- path-style toggle

### Works on mobile but not on desktop

This usually means the provider has unusual compatibility or networking behavior. Re-test after saving the exact same config on desktop. If needed, try a provider-known endpoint format rather than a generic one.

### Books appear but do not open immediately

The metadata may already be synced while the full book file is still remote. Open the book again and let ReadAny download it locally.

### Vectors do not appear on another device

This is expected. Vector indexes remain local to each device.
