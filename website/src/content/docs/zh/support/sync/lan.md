局域网快传是给 **一次性迁移** 用的，不是长期自动同步。

适合这些场景：

- 从旧设备迁到新设备
- 两台设备在同一个局域网里
- 你想快速导入数据，不想先配 WebDAV 或 S3

## 它是怎么工作的

一台设备作为 **发送方**。

另一台设备作为 **导入目标**。

导入目标会在局域网内连接发送方，并拉取：

- 数据库快照
- 书籍正文
- 封面

这次导入会覆盖目标设备上的对应数据。

## 推荐流程

### 在源设备上

1. 打开 **设置 → 同步 → LAN**
2. 选择 **发送此设备数据**
3. 启动服务
4. 保持应用打开

### 在目标设备上

1. 打开 **设置 → 同步 → LAN**
2. 选择 **导入另一台设备数据**
3. 扫描二维码，或手动输入 IP、端口和配对码
4. 确认覆盖提醒
5. 开始导入

## 重要说明

- 局域网快传是 **单向导入**
- 更适合迁移，不适合做合并
- 它不能替代 WebDAV 或 S3 的长期多设备同步
- 自动同步和同步间隔不适用于这个模式

## 桌面端和移动端的差异

- 移动端在支持时可以直接扫码
- 桌面端通常更适合手动输入连接信息
- 两边都必须在同一个局域网中

## 排障

### 无法连接到发送方

优先检查：

- 两台设备是否在同一局域网
- 发送方应用是否还开着
- 配对码是否一致
- IP 和端口是否正确

### 导入完成后仍有书缺失

重新尝试一次，并确保发送方设备在整个传输期间保持唤醒，应用不要退出。

### 能不能把 LAN 一直开着当长期同步用？

技术上可以一直开着，但这不是它的设计目标。长期多设备同步建议用 WebDAV 或 S3。
