ReadAny 支持基于 S3 兼容对象存储的长期多设备同步。

## 需要填写什么

在 **设置 → 同步 → S3** 中，通常需要填写：

- **Endpoint**
- **Region**
- **Bucket**
- **Access Key ID**
- **Secret Access Key**
- **Path-style** 开关（仅当服务商要求时）

[图片路径：S3 同步设置页]

## 字段说明

### Endpoint

也就是 S3 兼容服务的地址，例如：

- `https://<account>.r2.cloudflarestorage.com`
- `https://s3.cn-east-1.qiniucs.com`
- `https://cos.ap-guangzhou.myqcloud.com`

### Region

填写服务商要求的 region。

- 有些服务商使用标准 AWS 风格 region
- 有些服务商会使用 `auto`

### Bucket

ReadAny 会把同步数据存到这个 bucket 里。

### Path-style

只有服务商明确要求 path-style 时才打开。

如果同一组配置在某个平台上能用、另一个平台上不行，优先复查这里。

[图片路径：S3 服务商配置示例]

## S3 会存什么

S3 同步可以存：

- 设备同步快照
- 书籍元数据
- 笔记和阅读统计
- 封面
- 书籍正文文件

## 推荐顺序

1. 先在一台设备上配置 S3
2. 先点测试连接
3. 保存配置
4. 手动触发一次同步
5. 再在另一台设备上填同样的 bucket 配置
6. 再同步一次，必要时使用全量下载

## 全量上传与全量下载

如果你想让某一端彻底覆盖另一端：

- 在数据更完整的设备上执行 **全量上传**
- 在目标设备上执行 **全量下载**

这通常也是清理第一次同步混乱状态的最快方法。

## 排障

### 测试连接失败

优先检查：

- endpoint
- region
- bucket 名
- Access Key ID / Secret Access Key
- path-style 开关

### 移动端能用，桌面端不行

这通常意味着服务商的兼容实现或网络行为比较特殊。先确认桌面端使用的是完全相同的配置，再检查 endpoint 和 path-style 是否正确。

### 书已经出现，但打不开

这通常表示元数据已经同步过来了，但正文文件还在远端。重新点开书，让 ReadAny 先把正文下载到本地。

### 向量没有跟着同步

这是预期行为。向量索引是本地数据，不跨设备同步。
