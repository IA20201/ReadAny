语义搜索可以让你按“意思”找段落，而不只是按关键词精确匹配。

## 它是怎么工作的

ReadAny 会结合：

- **向量相似度检索**
- **关键词搜索**

所以相比普通全文搜索，通常能找到更相关的结果。

## 使用前要做什么

一本书需要先完成向量化。

通常流程是：

1. 打开一本书
2. 找到向量化入口
3. 选择本地或远程向量模型
4. 等待索引完成

## 本地向量与远程向量

### 本地向量

- 在当前设备上运行
- 向量索引保留在本地
- 在低性能设备上可能会更慢

### 远程向量

- 使用远程 API 做 embedding
- 在移动端尤其有用
- 处理完成后，索引仍然保存在本地

## 一个很重要的同步说明

向量索引是本地数据，不跨设备同步。

这意味着：

- 向量文件不会自动同步到其他设备
- 另一台设备即使已经拿到了同一本书，也可能还没有向量
- 同步不应该覆盖掉这台设备已经存在的本地向量

## 如何得到更好的结果

- 尽量用自然语言查询，不要只输几个孤立关键词
- 可以试试简短的意图式搜索，比如“作者关于劳动价值的论证”
- 如果你换了 embedding 模型或向量设置，必要时重新向量化

## 排障

### 搜索没有结果

- 确认这本书已经完成向量化
- 检查向量模型是否正确配置
- 尝试换一个更自然、更宽泛的查询

### 一台设备有向量，另一台没有

这是正常现象，因为向量不会跨设备同步。
