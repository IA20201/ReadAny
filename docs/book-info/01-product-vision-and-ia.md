# Product Vision And Information Architecture

## 核心定义

ReadAny 的“书籍信息”不应该只是一个展示标题、作者、简介的详情页。

它应该是一本书的主页，也是一块操作中枢：

- 进入前：帮助用户判断这本书值不值得读。
- 阅读中：帮助用户继续读、继续听、继续记。
- 读完后：帮助用户回看、整理、沉淀和分享。

换句话说，这个页面要把“书的客观信息”和“用户与这本书的主观关系”放在同一层里。

## 对标产品的启发

下面这些产品不是要照搬，而是借它们各自最强的点：

- `Kindle`
  - 强在阅读中的即时可用性：词典、X-Ray、Page Flip、My Notebook、剩余阅读时间、字体与版式控制。
  - 参考：[Amazon Kindle: Reading App](https://apps.apple.com/us/app/amazon-kindle/id302584613)
- `Apple Books`
  - 强在信息层级克制清晰：阅读目标、阅读记录、批注入口、分享成就。
  - 参考：[Set reading goals in the Books app on iPhone](https://support.apple.com/en-afri/guide/iphone/iph6013e96f4/26/ios/26)
  - 参考：[Annotate books in the Books app on iPhone](https://support.apple.com/en-al/guide/iphone/iph17bf340c1/ios)
- `Goodreads`
  - 强在评分、书评、想读/在读/读过、书架体系、推荐和社会证明。
  - 参考：[Goodreads: Book Tracker & More](https://apps.apple.com/us/app/goodreads-book-reviews/id355833469)
- `Kobo`
  - 强在标注资产化：笔记列表、导出标注、多格式导出。
  - 参考：[Export annotations from your books](https://help.kobo.com/hc/en-us/articles/29991333812631-Export-annotations-from-your-books)
  - 参考：[Make notes while you read in your Kobo Books app](https://help.kobo.com/hc/en-us/articles/23234736728215-Make-notes-while-you-read-in-your-Kobo-Books-app-iOS-Android)
- `StoryGraph`
  - 强在把阅读偏好结构化：mood、pace、page number、format、rating、阅读趋势。
  - 参考：[The StoryGraph stats](https://app.thestorygraph.com/stats/_tab)

我们的目标是把这几类能力融合成“阅读产品里的书籍主页”，而不是做成 Goodreads 那种很吵的社区书页。

## 核心模块

建议把书籍信息系统拆成五个标签页。

### 1. 概览

回答“这是什么书”“为什么值得读”。

包含：

- 封面 Hero
- 标题、副标题、作者、译者
- 标签、题材、主题、关键词
- 简介
- 目录预览
- 系列信息
- 相关推荐

### 2. 阅读

回答“我现在读到哪了”“这本书对我来说处于什么状态”。

包含：

- 阅读状态：未读 / 在读 / 读完 / 搁置 / 弃读 / 重读中
- 进度、当前位置、最后阅读时间
- 阅读时长
- 阅读字数
- 阅读速度
- 预计读完时间
- 阅读时间线
- 这本书在你阅读史中的特殊位置

### 3. 笔记

回答“我在这本书里留下了什么”。

包含：

- 高亮、笔记、书签
- 按章节/按时间/按颜色筛选
- 金句流
- 导出与分享
- AI 整理笔记

### 4. 洞察

回答“这本书在讲什么”“我为什么会喜欢它”。

包含：

- AI 摘要
- 人物 / 概念 / 主题卡
- 高频关键词
- 阅读重心
- 书籍 DNA
- 适合谁读 / 适合什么时候读

### 5. 版本

回答“这本书的文件和元数据状态如何”。

包含：

- 格式、来源、文件路径、大小
- 元数据来源
- 同步状态
- 封面来源
- 修复元数据
- 替换封面
- 多封面管理

## 信息架构原则

### 原则 1：先给行动，再给资料

最先看到的必须是：

- 继续阅读
- 听书
- 问 AI
- 评分 / 写评

而不是把简介和 ISBN 放最前面。

### 原则 2：把“书籍信息”和“我的关系”并列展示

很多阅读产品的详情页只讲书，不讲用户自己。

ReadAny 应该在首屏同时展示：

- 书本的基础信息
- 用户当前进度
- 用户沉淀的资产

### 原则 3：页面应该“常进常新”

如果这页只在导入后看一次，它就失败了。

要让用户愿意经常回来，必须加入这些动态内容：

- 最近高亮
- 阅读趋势
- 本书目标
- 评分、书评
- AI 洞察
- 封面集与个性化封面

## 移动端方案

移动端建议做成“沉浸式书籍主页”。

### 首屏结构

1. 顶部 Hero
   - 封面
   - 标题 / 作者
   - 状态胶囊
   - 进度胶囊
2. 主操作区
   - `继续阅读`
   - `听书`
   - `问 AI`
3. 横向标签
   - 概览 / 阅读 / 笔记 / 洞察 / 版本

### 移动端交互建议

- 上滑后封面缩成 sticky 头部
- 点封面进入“封面集”浏览
- 点状态胶囊快速切换 `想读 / 在读 / 读完 / 搁置`
- 点评分直接弹出评分面板
- 长按高亮卡片直接分享

### 移动端视觉原则

- 不做卡片套卡片
- 信息分区大块留白
- 主 CTA 明确
- 封面、评分、标签、笔记资产形成视觉节奏

## 桌面端方案

桌面端适合做成“双栏 + 固定左栏”。

### 左栏

- 大封面
- 标题 / 作者 / 标签
- 继续阅读
- 听书
- 问 AI
- 状态、评分、书评入口
- 快速统计

### 右栏

- 概览
- 阅读
- 笔记
- 洞察
- 版本

### 桌面端额外能力

- 阅读器内右侧抽屉版
- 书库 hover 快速预览
- 多封面选择器可展开为网格
- 笔记列表与章节树可以并列

## 富有记忆点的“好玩”能力

### 1. 你与这本书

做成一块关系卡：

- 第一次打开：哪天
- 最沉浸的一次：哪天，多久
- 最常阅读时段
- 最常记笔记的章节

### 2. 书籍 DNA

给每本书一组可视化标签：

- 节奏：慢 / 中 / 快
- 难度：轻 / 中 / 深
- 情绪：治愈 / 冷峻 / 热烈 / 压抑
- 结构：概念驱动 / 人物驱动 / 情节驱动

### 3. 一句话记忆

自动从高亮、笔记和摘要里提炼：

- “你最可能记住的一句话”

### 4. 阅读回放

时间线回看：

- 哪几天推进最快
- 哪章停留最久
- 什么时候开始认真读

### 5. 书籍海报

可分享的不仅是金句。

还可以分享：

- 我与这本书海报
- 封面集海报
- 评分与书评卡
- 本书阅读战报

## 首版推荐切入点

最值得第一批就做的能力：

1. 封面 Hero + 继续阅读 + 听书 + 问 AI
2. 状态切换
3. 评分与简短书评
4. 阅读统计和本书目标
5. 高亮 / 笔记 / 书签聚合
6. 版本信息与封面编辑

这些能力一旦搭起来，“书籍信息”就已经不是一个资料页，而是一本书的中枢页面。
