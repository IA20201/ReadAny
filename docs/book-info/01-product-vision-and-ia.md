# Product Vision And Information Architecture

## 核心定义

ReadAny 的"书籍信息"不应该只是一个展示标题、作者、简介的详情页。

它应该是一本书的主页，也是一块操作中枢：

- 进入前：帮助用户判断这本书值不值得读。
- 阅读中：帮助用户继续读、继续听、继续记。
- 读完后：帮助用户回看、整理、沉淀。

换句话说，这个页面要把"书的客观信息"和"用户与这本书的主观关系"放在同一层里。

## 对标产品的启发

下面这些产品不是要照搬，而是借它们各自最强的点：

- `Kindle`
  - 强在阅读中的即时可用性：词典、X-Ray、剩余阅读时间、字体与版式控制。
- `Apple Books`
  - 强在信息层级克制清晰：阅读目标、阅读记录、批注入口。
- `Goodreads`
  - 强在评分、想读/在读/读过、书架体系。
- `Kobo`
  - 强在标注资产化：笔记列表、导出标注。

我们的目标是把这几类能力融合成"本地阅读产品里的书籍主页"。ReadAny 是本地单机应用，不做社区，所以 Goodreads 式的社会证明和推荐不在范围内。

## 核心模块

MVP 阶段建议拆成三个标签页，后续视内容丰富度再扩展。

### 1. 概览

回答"这是什么书""基本情况如何"。

包含：

- 标题、副标题、作者、译者
- 标签
- 简介
- 目录预览
- 文件信息（格式、大小、来源、路径）
- 同步状态

> 注：文件/版本信息不单独成 Tab，折叠在概览底部即可，因为早期内容太少不值得独立页面。

### 2. 阅读

回答"我现在读到哪了""我和这本书什么关系"。

包含：

- 阅读状态：未读 / 在读 / 读完 / 搁置 / 弃读
- 进度、当前位置、最后阅读时间
- 阅读时长（来自 `ReadingSession` 累计）
- 阅读字数（来自 `charactersRead` 累计）
- 阅读速度（字/分钟）
- 预计读完时间（来自 `eta-service`）
- 本书阅读时间线（按天的阅读量）

### 3. 笔记

回答"我在这本书里留下了什么"。

包含：

- 高亮（highlights 表）
- 笔记（notes 表）
- 书签（bookmarks 表）
- 按章节/按时间筛选
- 导出

> 后续扩展：当 AI 能力成熟后，可以新增"洞察"Tab（AI 摘要、人物/概念卡、关键词提炼）。但 MVP 不做，因为内容会很空。

## 信息架构原则

### 原则 1：先给行动，再给资料

最先看到的必须是：

- 继续阅读
- 听书
- 问 AI

而不是把简介和 ISBN 放最前面。

### 原则 2：把"书籍信息"和"我的关系"并列展示

很多阅读产品的详情页只讲书，不讲用户自己。

ReadAny 应该在首屏同时展示：

- 书本的基础信息（标题、作者、封面）
- 用户当前进度与阅读时长
- 用户评分

### 原则 3：页面应该"常进常新"

如果这页只在导入后看一次，它就失败了。

要让用户愿意经常回来，必须加入动态内容：

- 阅读进度变化
- 最近高亮
- 阅读趋势
- 评分/短评

## 移动端方案

移动端做成"沉浸式书籍主页"，一个完整页面。

### 首屏结构

1. 顶部 Hero
   - 封面（点击可替换）
   - 标题 / 作者
   - 阅读状态胶囊
   - 进度胶囊
   - 评分（星级）
2. 主操作区
   - `继续阅读`（一号按钮）
   - `听书`
   - `问 AI`
3. 横向标签
   - 概览 / 阅读 / 笔记

### 移动端交互建议

- 上滑后封面缩成 sticky 头部
- 点封面可替换主封面
- 点状态胶囊快速切换 `未读 / 在读 / 读完 / 搁置 / 弃读`
- 点评分直接弹出评分面板

## 桌面端方案

桌面端做成"双栏 + 固定左栏"，一个完整页面。

### 左栏

- 大封面（点击可替换）
- 标题 / 作者 / 标签
- 继续阅读 / 听书 / 问 AI
- 阅读状态
- 评分 + 一句话短评
- 快速统计（时长、进度、字数）

### 右栏

- 概览 / 阅读 / 笔记（三个 Tab）

> 后续扩展：阅读器内右侧抽屉版、书库 hover 快速预览。这些等完整页面做稳后再做，避免同时维护多套布局。

## 阅读状态的状态机

导入和阅读过程中，状态需要有明确的自动流转规则：

| 触发事件 | 状态变化 |
|---|---|
| 导入新书 | → `未读` |
| 首次打开阅读器 | → `在读` |
| 进度达到 100% | 弹窗询问是否标记为 `读完` |
| 用户手动切换 | 任意状态之间可手动切换 |
| 再次打开已标记"读完"的书 | 保持 `读完`，不自动变回"在读" |

> "重读"不做独立状态。如果用户想重读，手动切回"在读"即可。减少状态数量，降低复杂度。

## 首版推荐切入点

最值得第一批就做的能力：

1. 封面 Hero + 继续阅读 + 听书 + 问 AI
2. 阅读状态切换（带自动流转）
3. 星级评分 + 一句话短评
4. 阅读统计摘要（时长、进度、字数、速度）
5. 高亮 / 笔记 / 书签聚合
6. 文件信息查看（只读）
7. 主封面替换（单张替换，不做封面集）

这些能力一旦搭起来，"书籍信息"就已经不是一个资料页，而是一本书的中枢页面。
