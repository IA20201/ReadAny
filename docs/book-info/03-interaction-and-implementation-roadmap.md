# Interaction And Implementation Roadmap

## 目标

先把"书籍信息"做成真正可用的功能，再慢慢把它做得越来越有意思。

路线要分层：

- 先做日常高频（阅读状态、评分、继续阅读）
- 再做编辑能力（元数据修正）
- 最后做进阶体验（AI 洞察、抽屉/预览形态）

## 入口

书籍信息最重要的入口：

1. 从书库点击一本书（长按或右键菜单中的"详情"）
2. 从阅读器顶部书名进入
3. 从笔记/高亮反查到书
4. 从统计页点击一本书
5. 从搜索结果进入

MVP 只做 **一种形态：完整页面**。

理由：

- 同时维护完整页面 + 侧边抽屉 + hover 预览 = 三套布局组件，维护成本翻三倍
- 完整页面做好了，抽屉版只是裁剪布局，迁移成本低
- 先把一种形态做到好用，再扩展

## 关键交互细节

### 1. 继续阅读必须是一号动作

无论移动端还是桌面端，最显眼的都必须是 `继续阅读`。

这个按钮应该根据状态变化：
- 未读 → `开始阅读`
- 在读 → `继续阅读`（显示进度）
- 读完 → `重新阅读`
- 搁置/弃读 → `重新开始`

### 2. 评分不能藏太深

评分与短评应该紧挨标题区，不要埋到更多菜单。

交互：
- 点击星星即可评分，无需额外确认
- 再次点击同一星级 = 清除评分
- 评分旁边显示"写一句话..."的灰色占位文字，点击展开输入

### 3. 封面替换要"轻"

不需要弹出复杂的封面管理器。

交互：
- 移动端：点封面 → ActionSheet（从相册选择 / 从文件选择 / 恢复原封面）
- 桌面端：点封面 → 文件选择对话框，右键封面 → 恢复原封面
- 替换后直接生效，覆写 `covers/{bookId}.{ext}`

### 4. 阅读状态切换

交互：
- 移动端：点击状态胶囊 → 底部弹出状态列表
- 桌面端：点击状态胶囊 → Dropdown 选择
- 切换后即时保存，无需确认

### 5. 笔记 Tab 的展示

优先按时间倒序展示，支持按章节分组切换。

每条笔记/高亮显示：
- 引用文本
- 用户批注（如果有）
- 所在章节
- 创建时间
- 颜色标记

长列表虚拟化，避免几百条笔记导致卡顿。

## 与现有系统的联动

### 与阅读器

- 从阅读器标题进入书籍信息
- 从书籍信息一键回到当前位置
- 打开阅读器自动更新 `readingStatus` 为 `'reading'`

### 与统计

- 本书阅读时长 = `reading_sessions` 表中该 `bookId` 的 `totalActiveTime` 求和
- 本书阅读字数 = `reading_sessions` 表中该 `bookId` 的 `charactersRead` 求和
- 阅读速度 = 字数 / 时长
- 预计读完时间 = 来自 `eta-service`

### 与笔记

- 直接查询 `highlights` / `notes` / `bookmarks` 表中 `bookId` 匹配的记录
- 复用现有的笔记组件

### 与 TTS

- 如果本书支持朗读，显示"听书"按钮
- 显示上次听到的位置
- 接入 `tts-store` 的播放状态

### 与 AI

- "问这本书"按钮 → 打开以本书为上下文的对话
- 需要该书已完成向量化（`isVectorized === true`）
- 未向量化时显示"向量化后可用 AI 问答"的提示

### 与书库

- 书库 `BookCard` 的右键菜单 / 长按菜单新增"查看详情"入口
- 书籍信息页的编辑（评分、状态、标签）实时反映到书库列表

## 分阶段实施

### Phase 0：冻结方案（当前）

产出：

- 本目录三份文档

目标：

- 明确信息架构
- 明确编辑边界
- 明确首版范围

### Phase 1：书籍信息 MVP

目标：

- 建立可访问的"书籍主页"

范围：

- 数据层：`books` 表新增 `reading_status` / `rating` / `short_review` 三个字段 + 迁移脚本
- `Book` 类型扩展 + `book-queries.ts` 更新
- 移动端 + 桌面端完整页面
- Hero + 继续阅读 / 听书 / 问 AI
- 概览 Tab（元数据 + 文件信息）
- 阅读 Tab（状态 + 进度 + 时长 + 字数 + 时间线）
- 笔记 Tab（高亮 + 笔记 + 书签聚合）
- 阅读状态切换（含自动流转）
- 星级评分 + 一句话短评
- 主封面替换

估算：~2 周

### Phase 2：可编辑元数据

目标：

- 让用户能修正不准确的书籍信息

范围：

- `meta_override` JSON 字段
- 标题、作者、简介的编辑 UI
- 编辑/恢复原值的交互
- 标签管理（复用现有）

估算：~1 周

### Phase 3：AI 洞察（视 AI 能力成熟度）

目标：

- 让这页开始有智能感

范围：

- 新增"洞察"Tab
- AI 摘要
- 关键概念提取
- 阅读建议

前提：需要 AI 输出质量足够好，否则不如不做。

### Phase 4：抽屉与预览形态

目标：

- 让书籍信息嵌入整个产品流程

范围：

- 阅读器右侧抽屉版
- 书库 hover 快速预览
- 搜索结果快捷信息

前提：完整页面已稳定运行 1+ 个版本。
