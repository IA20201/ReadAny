# Period Reports And Sharing

## 总原则

`日 / 周 / 月 / 年 / 总` 不应该只是一个筛选器，而应该是五种不同的报告形态。

前四种都是“可导航周期”。

- 日：上一天 / 下一天 / 指定日期
- 周：上一周 / 下一周 / 指定周
- 月：上一月 / 下一月 / 指定月
- 年：上一年 / 下一年 / 指定年
- 总：固定全生命周期，不切时间段

## 一、日报告

### 目标

回答“今天读得怎么样”。

### 页面重点

- 今日总时长
- 今日 session 数
- 今日页数
- 今日读了几本书
- 最长一次阅读
- 今天首次 / 最后一次阅读时间
- 今天主要阅读时段
- 今天主读书籍
- 与昨天相比的变化

### 建议模块

- Hero：`今天读了 48 分钟`
- 时间轴：按小时分布
- 书籍块：今天读过的书
- 对比块：较昨天 `+18%`
- 习惯洞察：`你通常在晚上阅读`

### 日分享图

推荐内容：

- 标题：`今日阅读报告`
- 周期：`2026年4月16日`
- 主指标：今日阅读时长
- 次指标：session 数、页数、主读书
- 图形：今日阅读时间轴
- Footer：`今天也读了一点点，已经很棒了`

## 二、周报告

### 目标

回答“这一周的阅读节奏是什么样的”。

### 页面重点

- 本周总时长
- 活跃天数
- 日均时长
- 最佳阅读日
- 工作日 / 周末阅读分布
- 本周 Top 书籍
- streak 延续情况

### 建议模块

- Hero：`这周连续读了 6 天`
- 周条形图：周一到周日
- 周洞察：最常阅读的星期几
- Top Books：本周投入最多的书
- streak 卡片

### 周分享图

- 标题：`本周阅读报告`
- 周期：`2026年第16周`
- 主指标：本周总时长
- 次指标：活跃天数、日均时长、Top 书
- 图形：一周 7 天条形图
- Footer：`这一周，你把阅读重新变成了习惯`

## 三、月报告

### 目标

回答“这个月你的阅读密度和产出如何”。

### 页面重点

- 月总时长
- 活跃天数 / 覆盖率
- 本月完成书数
- 月热力图
- 每周阅读分布
- 本月最佳一天
- 本月 Top 书 / 作者 / 分类

### 建议模块

- Hero：`这个月读了 17 小时`
- 热力图：整月按天分布
- 周节奏：四周或五周分布
- 完成书籍卡片
- 月度回顾文案

### 月分享图

- 标题：`本月阅读报告`
- 周期：`2026年4月`
- 主指标：本月总时长
- 次指标：阅读天数、完成书数、最佳一天
- 图形：月热力图或周趋势
- Footer：`四月，你让阅读变得更稳定了`

## 四、年报告

### 目标

回答“这一年你的阅读趋势和结构是什么”。

### 页面重点

- 年总时长
- 年阅读天数
- 月趋势图
- 最强月份
- 年度完成书数
- Top 作者 / Top 分类
- 年度最长 streak

### 建议模块

- Hero：`这一年读了 126 小时`
- 月趋势图：1-12 月
- 年度最强月份
- 年度阅读画像
- 年度里程碑

### 年分享图

- 标题：`年度阅读报告`
- 周期：`2026年`
- 主指标：全年总时长
- 次指标：阅读天数、完成书数、最强月份
- 图形：月趋势折线或柱状图
- Footer：`这一年的你，读得越来越像自己了`

## 五、总报告

### 目标

回答“你和 ReadAny 一起走了多久，以及你累计成了什么样的人”。

### 页面重点

- 遇到 ReadAny 多久了
- 生涯总时长
- 生涯总书数
- 生涯总阅读天数
- 生涯总 session
- 最长 streak
- 历史 Top 书 / 作者
- 生涯时间线 / 里程碑

### 陪伴感文案

这里一定要有人味：

- `ReadAny 已陪你 428 天`
- `我们一起读过 216 小时`
- `你已经把阅读坚持成一种生活方式`

### 建议模块

- Hero：`已经陪你 428 天`
- 生涯累计卡片
- 里程碑时间线
- 历史 Top 列表
- 你的阅读画像

### 总分享图

- 标题：`我的阅读旅程`
- 周期：`从 2024年8月开始`
- 主指标：陪伴天数
- 次指标：总时长、总书数、最长 streak
- 图形：里程碑或年趋势
- Footer：`ReadAny 陪你读到现在`

## 周期导航统一方案

五个维度顶部统一为：

```text
[维度切换]   [←] [周期标签] [→]   [分享]
```

示例：

- 日：`[日][周][月][年][总]  ←  2026年4月16日  →`
- 周：`[日][周][月][年][总]  ←  第16周 · 4/14 - 4/20  →`
- 月：`[日][周][月][年][总]  ←  2026年4月  →`
- 年：`[日][周][月][年][总]  ←  2026年  →`
- 总：`[日][周][月][年][总]  从 2024年8月 开始`

## 分享能力统一方案

每个维度都支持分享，但分享不是页面截图，而是“图片卡片生成”。

### 支持形式

- 保存图片
- 系统分享
- 复制到剪贴板

### 推荐画布规格

- 标准竖版：1080 x 1440
- 小红书 / 朋友圈适配版：1242 x 1660
- 横版备用：1600 x 900

### 视觉结构

统一结构：

1. Header：标题 + 周期
2. Hero Metric：主指标
3. Secondary Metrics：2-4 个次指标
4. Chart / Heatmap / Timeline
5. Top Book / Insight
6. Footer：品牌 + 一句总结

### 品牌风格建议

- 默认沿用 ReadAny 当前暖色调
- 后续接入主题系统后，分享图也可套主题

## 移动端与桌面端的差异

报告内容一致，但布局不同。

- 桌面端：
  - 可以左右双栏
  - 图表和列表同时展示
  - 适合年 / 总这类大报告

- 移动端：
  - 纵向卡片流
  - Hero 在最前
  - 图表、书籍列表、洞察依次展开
  - 分享按钮固定在顶部或底部浮层

## 不建议现在做的事情

- 不要一开始就做“用户自定义分享模板”
- 不要先做 DOM 截图方案
- 不要把图表配置塞进页面组件内部

先把每个维度的 `shareCard model` 定下来，后面无论换 UI 还是换分享模板都不怕。
